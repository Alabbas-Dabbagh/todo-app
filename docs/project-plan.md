# Project Plan & Management

## Project
Mobile task manager (To-Do App)

## Author
Alabbas Dabbagh

---

# Development Approach

The project was developed following the Software V-Model structure, combined with a simple agile sprint approach.

---

# Phase 1 – Requirements Analysis

- Definition of functional and non-functional requirements
- Documentation in `requirments.md`
- Clear acceptance criteria for each requirement

---

# Phase 2 – Architecture Design

- Definition of system components
- UI layer
- State management
- Persistence layer (AsyncStorage)
- UML documentation created

---

# Phase 3 – Implementation

Sprint 1:
- Project setup (Expo, GitHub)
- Basic UI structure

Sprint 2:
- Add / Toggle / Delete functionality
- AsyncStorage integration

Sprint 3:
- Styling improvements
- Code commenting
- Statistics dashboard (Explore tab) and summary cards on the main screen
- Final testing

---

# Phase 4 – Testing

- Manual system tests for each requirement
- Test documentation in `system-tests.md`
- Verification of persistence functionality

---

# Tools Used

- React Native
- Expo
- TypeScript
- AsyncStorage
- Git & GitHub
- Visual Studio Code

---

# Version Control Strategy

- Feature-based commits
- Separate commits for documentation, implementation and styling
- Remote repository on GitHub
